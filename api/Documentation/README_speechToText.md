# Speech-to-Text API Documentation

This module handles audio recording and interaction with the Groq Whisper API to convert spoken voice into text.

## ⚙️ Features

* **Voice Recording:** Captures audio from the microphone and saves it to a temporary file.
* **AI Transcription:** Sends the audio file to the Groq `whisper-large-v3` model for transcription.
* **Automatic Cleanup:** Deletes temporary files after processing to conserve disk space.
* **Error Handling:** Returns clear JSON responses with appropriate HTTP status codes (200 for success, 500 for error).

---

## 🛠 Functions & Components

### 1. `getRecording(sec=5)`

Records audio from the microphone and saves it as a temporary `.wav` file.

* **Parameters:**
  * `sec` (int, default: `5`) – The recording duration in seconds.
* **Returns:**
  * `str` – The path to the temporary generated file (e.g., `voice_abcdef123456.wav`).

### 2. `speechToText(audio)`

Sends the recorded audio file to the Groq API and returns the transcribed text.

* **Parameters:**
  * `audio` (str) – The path to the audio file generated by `getRecording`.
* **Returns:**
  * `str` – The transcribed text from the Whisper model.
* **Possible Errors:**
  * Raises an exception if the file does not exist or if the Groq API connection fails.

---

## 🔌 Endpoints

### `POST /speechToText/`

Triggers the recording and transcription process.

#### Responses:

* **Success (`200 OK`)**
  The application records for 10 seconds and returns the transcribed text:

  {
    "status": "success",
    "transcription": "Hello, how can I help you today?"
  }
